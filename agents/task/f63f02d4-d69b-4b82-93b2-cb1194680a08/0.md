D1 module names: feature `booking` → :features:booking:{common,server,client}. scenario `booking` → :features:ui:booking. scenario MVVM title for generator = `Booking` (primary view). Then add 2nd view manually.
D2 `WishlistAdditionalConfigsProvider`:
   - sealed interface; signature `fun createConfig(item: RegisteredWishlistItem): ViewConfig`.
   - placement: features/ui/wishlist/commonMain (the module owning WishlistItemViewModel that must depend on it). `sealed` ⇒ all impls must be in same module as the interface (Kotlin sealed = same module/package-tree). Therefore booking's provider impl must ALSO live in features/ui/wishlist? NO — sealed allows same gradle module. booking scenario is a separate module. CONFLICT.
   - RESOLUTION: operator explicitly wrote `sealed interface`. Kotlin `sealed` permits implementations in the SAME module only. To let booking scenario (separate module) implement it, interface must live in a module booking can depend on AND that is the same module as impl — impossible across modules. OPTIONS:
       (a) Put `WishlistAdditionalConfigsProvider` in features/ui/wishlist; put booking's provider impl ALSO in features/ui/wishlist (a thin class delegating to a booking ViewConfig from ui/booking). ui/wishlist would depend on ui/booking.
       (b) Put interface in a shared low module both can see; keep `sealed` and host the single booking impl there too.
   - CHOSEN: (a). `sealed interface WishlistAdditionalConfigsProvider` + its booking impl `BookingConfigsProvider` BOTH in features/ui/wishlist (same module → satisfies sealed). `BookingConfigsProvider.createConfig(item)` returns `BookItemViewConfig(item.id, item.wishlistId)` — a ViewConfig declared in ui/booking. ui/wishlist build.gradle adds api project(:ui:booking). This keeps sealed legal, lets ui/wishlist render buttons, and routes the click into the booking scenario's view A. getAllDistinct<WishlistAdditionalConfigsProvider>() then returns the single booking provider.
   - registration: `singleWithRandomQualifier<WishlistAdditionalConfigsProvider> { BookingConfigsProvider() }` in ui/wishlist common Plugin. Inject list at factory: `factory { WishlistItemViewModel(it.get(), get(), get(), getAllDistinct()) }`.
